Verification API Documentation

Verification API is owned by SitePhantom Development

Authentication & Access

This API is not publicly open. Access is restricted to:

  • DeltaDB Cloud Services
  • Authorized SitePhantom Development Endpoints
  • Registered Developers

No API keys are issued externally. All requests must originate from approved sources.

⚠️ Unauthorized requests will be blocked silently or return 403 Forbidden.

Endpoints

GET /public/verification-database

Pull the full database of verified accounts with SitePhantom Development.

Request Body (JSON):

{
  "apikey": "YOUR_API_KEY"
}

Success Response (200 OK):

{
  "success": true,
  "developer": true,
  "data": "VERIFICATION_DATABASE_CONTENT"
}

Error Response (400 or 403):

{
  "success": false,
  "error": "invalid_key"
}

GET /public/mass-user-database/pull

Check verification status of a user (requires trusted source).

Example: GET /public/mass-user-database/pull?apikey=YOUR_API_KEY

Response:

{
  "success": true,
  "developer": true,
  "data": "MASS_USER_DATABASE_CONTENT"
}

⏱Rate Limiting

All endpoints are protected by DeltaDB Cloud Services:

  • 30 requests per minute per IP
  • Exceeding limits returns 429 Too Many Requests

Repeat overusing the API may result in your access being revoked.

Integration Tips

  • When pulling the mass-user-database, ensure your drive has enough storage, as it is a quite large database.
  • Handle 4xx/5xx responses gracefully in your client.
  • Prevent your users from overusing your service that uses this API, as your apikey can get revoked.
  • Implement securely and don't leak your API key.
